# 專案工作規則

- 每次回應使用者問題或執行任務前，都要先讀取 `AGENTS.md`、`問題解決紀錄.md` 與 `未解決.md`。
- 每次完成使用者提出的問題或任務後，都要更新 `問題解決紀錄.md`。
- 紀錄內容至少包含：日期、問題、解法、結果。
- 如果任務尚未解決，不要寫成已解決；可以在回覆中說明阻礙。
- 不要刪除原始檔；如需移除或覆蓋既有檔案，必須先取得使用者明確同意。
- 回覆使用者時請使用繁體中文。
- 修改檔案前，先說明準備修改哪個檔案與修改內容。
- 任務完成後，回覆需附上處理摘要。
- 如果任務或問題尚未解決，要記錄到 `未解決.md`，並說明目前狀態與待處理事項。
- 每次修改並更新本地 Markdown 檔案（如 `AGENTS.md`、`問題解決紀錄.md`、`未解決.md` 等）後，必須自動執行 Git commit 與 push，同步至 GitHub 遠端儲存庫。
